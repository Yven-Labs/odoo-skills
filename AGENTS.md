# odoo-skills — Code Review Rules

## Context

This repository contains the **Odoo AI Skills Ecosystem** — a library of AI agent skills
for Odoo development, covering v16, v17, v18, and v19.

### Repository Structure

```
odoo-skills/
├── PRD.md                  — Product Requirements Document
├── README.md               — Project overview
├── AGENTS.md               — Root skill router (auto-generated by sync.sh)
├── backend/AGENTS.md       — Backend skill router
├── frontend/AGENTS.md      — Frontend skill router
├── devops/AGENTS.md        — DevOps skill router
├── setup.sh                — Multi-tool installer (macOS/Linux)
├── setup.ps1               — Multi-tool installer (Windows)
└── skills/
    ├── skill-sync/         — Auto-sync system for AGENTS.md
    ├── odoo/               — Overview skill
    ├── odoo-module/        — Module structure skill
    ├── odoo-orm/           — ORM patterns skill
    ├── odoo-views/         — XML views skill
    ├── odoo-security/      — Security patterns skill
    ├── odoo-testing/       — Testing patterns skill
    └── ...                 — More skills
```

---

## Universal Rules (all files)

REJECT if:
- Hardcoded secrets, tokens, API keys, or credentials
- TODO/FIXME left without an associated issue reference
- Files committed without a clear purpose (debug files, temp files)

---

## Markdown / Skill Files (*.md, SKILL.md)

REJECT if:
- Code blocks missing language identifier (` ```python `, ` ```xml `, etc.)
- Broken relative links to files that don't exist
- Instructions that only cover one OS without noting platform differences
- SKILL.md missing required frontmatter (`name`, `description` with Trigger, `metadata`)
- SKILL.md missing `metadata.scope` or `metadata.auto_invoke` fields
- Version-specific content in SKILL.md body instead of `assets/v{N}_patterns.md`

PREFER:
- Version-agnostic content in SKILL.md (80%), version deltas in assets/
- Tables for decision trees and version comparisons
- Minimal, focused code examples
- Explicit "WORKS ON: v{N}" and "DOES NOT WORK ON: v{M}" markers

---

## Shell Scripts (*.sh)

REJECT if:
- Missing `set -euo pipefail` at the top
- Unquoted variables: `$VAR` instead of `"$VAR"`
- Hardcoded absolute paths
- Missing OS detection when behavior differs between macOS/Linux

PREFER:
- `command -v tool` over `which tool`
- Colored output with named variables (`RED`, `GREEN`, `NC`)
- `mktemp` for temporary files

---

## PowerShell Scripts (*.ps1)

REJECT if:
- Missing `#Requires -Version 5.1`
- `$ErrorActionPreference` not set to `"Stop"`
- Hardcoded paths instead of `$env:USERPROFILE`

PREFER:
- `Join-Path` over string concatenation for paths
- `Test-Path` before file operations

---

## Contribution Standards

REJECT if:
- Commit does not follow Conventional Commits: `type(scope): description`
  - Valid types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`
  - Valid scopes: skill name (e.g., `odoo-orm`), `infra`, `sync`, `setup`
- New skill added without running `sync.sh` and verifying AGENTS.md updated
- New skill missing version-specific assets when the skill has version differences
- Code template added without testing on at least Odoo 17 or 18
